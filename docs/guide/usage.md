# SpecViewer の使い方

SpecViewer は設計書（仕様書）を閲覧・検索・ナビゲートするための社内ポータルです。この文書では、閲覧者向けの操作と、設計者向けの文書追加方法を説明します。

## 画面構成

| 画面 | 役割 |
|---|---|
| **ホーム** | セクションカード（API / 画面 / ガイド / DB）と最近の更新 |
| **セクション一覧** | フィルタ（ステータス / サブカテゴリ / 並び順）＋ 文書リスト |
| **詳細** | 本文 ＋ サイドバー（セクションツリー）＋ 目次 ＋ 前後の文書 |
| **検索パレット** | ⌘K で起動する全文検索 |
| **依存関係マップ** | 文書の依存（`dependsOn`）と被依存をグラフで可視化 |

## 閲覧者向けの操作

### 検索（⌘K / Ctrl+K）

- ヘッダーの検索バー、または `⌘K`（Mac）/ `Ctrl+K`（Windows）で検索パレットを開く
- キーワードを入力すると、文書のタイトル・説明・サブカテゴリから部分一致で検索
- 上部タブでセクション（API / 画面 / ガイド / DB）を絞り込み
- `↑` `↓` で選択、`↵` で開く、`esc` で閉じる

### ナビゲーション

- **グローバルナビ**（ヘッダー）: セクションを切り替える。現在のセクションがハイライト
- **パンくず**: 現在位置（セクション › サブカテゴリ › 文書）を確認。クリックで上の階層へ
- **サイドバー**: 同じセクション内の文書ツリー。文書をクリックで移動
- **前後の文書**: 詳細画面の下部に「前」「次」の文書リンク

### サイドバー

- **サブカテゴリの折たたみ**: サブカテゴリ見出しの `▾` をクリックで展開/収納（状態は記憶）
- **全体の折たたみ**: サイドバー右上の `«` ボタン、または `Ctrl`/`Cmd` + `B`。隠した後は左端の `»` で復元

### ダークモード

- ヘッダーの **月 / 太陽アイコン** で切り替え。初回は OS 設定に追従し、手動切り替えは記憶される

### 依存関係マップ（`/deps/`）

- 左のセレクタ、またはグラフの**ノードをクリック**で中心の文書を切り替え
- 矢印 `A → B` は「A が B に依存（A の前提は B）」
- **深さ: 2** を選ぶと、間接依存（依存の依存）を点線ノードで追加表示
- **範囲** でセレクタを特定セクションに絞り込み

## 設計者向け: 文書の追加

文書は Markdown（`docs/{api,ui,guide,db}/.../*.md`）で追加します。ビルド時に自動で全画面に反映されます。

### frontmatter

```yaml
---
title: 注文 API 仕様                # 必須: 文書タイトル
description: エンドポイント・リクエスト・レスポンス形式  # 一覧・OGP で使用
subcategory: 注文                  # 必須: サブカテゴリ（サイドバーツリーの分類）。文字列1つ または 配列で多階層（例: [注文, 決済連携]）
status: 承認済み                   # 必須: ドラフト | レビュー中 | 承認済み
dependsOn:                         # 任意: 依存している文書（docs/ からの path）
  - api/payment/payment-api
related:                           # 任意: 関連文書（docs/ からの path）
  - api/order/order-state
---

# 本文
...
```

### ポイント

- **section** はディレクトリの第1階層（`docs/api/` → API）から自動判定。**既存のセクション（API/画面/ガイド/DB）への文書追加**なら、`docs-data.ts` やコンポーネントを触らず `.md` を置くだけで全画面に反映されます
- **更新日時** は frontmatter に書かず、**git の最終コミット日時**から自動取得します。コミットすれば反映
- **`dependsOn` / `related`** は docs/ からの相対 path（`.md` 無し）で指定。title でなく path なので、title を変えてもリンクが切れません
- 本文の見出し（`##`）は、詳細画面右側の目次（TOC）に自動反映されます

> **新しい大カテゴリを追加する場合**（API/画面/ガイド/DB 以外の第5セクションなど）はコードの追記が必要です — `docs-data.ts`（`SectionKey` 型 + `sections` 配列）と `SectionIcon.vue`（アイコン）の2ファイル。手順は `DESIGN.md` §14「大カテゴリ（セクション）の追加手順」にまとめてあります。

詳しくは [執筆ガイドライン](./writing) を参照してください。
