# API設計フォーマット

「API」セクションに登録する設計書（**原則 1文書＝1エンドポイント**）の標準フォーマットを定める。
**テンプレート（セクション構成）** と **記述規約** の2層で運用する。

::: tip 方針
- 人間可読の **Markdown のみ** で記述する（OpenAPI は使用しない）
- 規範サンプルは [`POST /signup`](../api/auth/signup.md) を参照
:::

## 1. テンプレート（セクション構成）

| セクション | 必須 | 内容 |
|---|:---:|---|
| API情報 | 必須 | メソッド・パス・説明 |
| リクエスト | 必須 | ヘッダー・ボディ・サンプル |
| レスポンス | 必須 | 成功時・エラー時 |
| 副作用 | 必須 | 内部状態・外部への副作用・共通の失敗時の扱い（該当なければ「なし」） |
| シナリオ | 必須 | 振る舞いを Gherkin（Given-When-Then）で記述 |

### 1.1 API情報

```
# API情報
### メソッド
POST
### パス
/signup
### 説明
新しいユーザーアカウントを作成し、認証用トークンを発行する。
```

### 1.2 リクエスト

- **ヘッダー** — `名前 / 必須 / 説明`
- **ボディ** — `フィールド / 型 / 必須 / 制約 / 説明`（※`制約` 列に文字数・形式等を明記）
- **サンプル** — JSON を必ず併記

### 1.3 レスポンス

- **成功時** — ステータス（例: `201 Created`）＋ `フィールド / 型 / 説明` ＋ JSON
- **エラー時** — `ステータス / コード / 説明` ＋ JSON

### 1.4 副作用

各副作用は **内容 / 経路 / 失敗時** の3項目で記述する（対象は見出しに示す）。項目の取捨選択をせず、常に同じ項目を埋める。

- **内容** — 何を書き込むか・何を送信するか（ハッシュ化等の処理も明記）
- **経路** — 実行経路（DB トランザクション内／同期／非同期 など）
- **失敗時** — リトライ・フォールバック・ロールバックなど、失敗時の扱い

区分は `###` 見出しで分ける：

- **内部状態（DB トランザクション内）** — 主操作と同一トランザクション
- **外部への副作用** — DB トランザクション外（commit 後）
- **共通の失敗時の扱い** — 以下4項目を固定で記述：
  - **整合性** — 主操作と副作用の成否分離の原則（最終的整合性など）
  - **主操作失敗時** — DB トランザクション失敗時の挙動
  - **リトライ** — 外部副作用のリトライ方針
  - **冪等性** — 冪等キーと重複防止

### 1.5 シナリオ（BDD / Gherkin）

`Feature` / `Background` / `Scenario` で振る舞いを記述。Cucumber 等の実行対象ではなく、**レビュー用の仕様** として扱う。正常系だけでなく、重複・制限・副作用失敗・冪等性のシナリオも含める。

## 2. 記述規約

| 項目 | 規約 |
|---|---|
| エラー形式（共通） | `{ "code", "message" }`（`details[]` は必要な場合のみ任意追加） |
| エラーコード | `UPPER_SNAKE_CASE`（例: `EMAIL_ALREADY_USED`） |
| 成功ステータス | `200` / `201` / `204` |
| エラーステータス | `400`(形式・バリデーション) / `401` / `403` / `404` / `409` / `429` / `500` |
| バージョニング | URLパス（`/v1/` 等） |
| 命名 | JSON=`camelCase`／パス=`kebab-case`／リソース=複数名詞 |
| 日時 | ISO 8601 UTC（`2026-06-14T09:00:00Z`） |
| ページネーション | 標準化せず **案件毎** に決定（offset / cursor） |
| 冪等性 | 副作用は idempotency key（例: `userId`）で重複を弾く |
| 副作用の失敗 | 主操作（DB確定後）の成否と分離して取り消さない（最終的整合性）。非同期リトライ・復旧不能時は運用通知 |

### 2.1 共通エラーレスポンス

```json
{
  "code": "EMAIL_ALREADY_USED",
  "message": "このメールアドレスは既に使用されています。"
}
```
