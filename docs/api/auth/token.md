# トークン仕様

認証で発行されるトークンペア（アクセス / リフレッシュ）の形式と有効期限を定義する。

## トークン種別

| 種別 | 用途 | 有効期限 |
|---|---|---|
| アクセストークン | API 呼び出しの認証 | 1 時間 |
| リフレッシュトークン | アクセストークンの再発行 | 14 日 |

## 形式

- アクセストークン: **JWT**（HS256 署名）。クレームに `sub` / `scope` / `exp` を含む。
- リフレッシュトークン: 不透明トークン（ランダム 32 バイト、base64url）。

## 失効

- アクセストークン: 短寿命のため失効リストを持たない。
- リフレッシュトークン: ローテーション（[セッション管理](./session) 参照）とユーザー操作での失効に対応。
